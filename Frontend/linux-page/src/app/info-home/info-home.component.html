Permisos UNIX
  • R (read) → 4
  • W (write) → 2
  • X (execute) → 1

Archivos
  • - → archivo plano
  • d → directorio
  • l → enlace
  • b → archivo de bloque
  • c → archivo de caracteres
  • p → archivo pipe
  • s → archivo socket

> ls -l

> - rwx r-- r-- 2 root root 320 dic 2 14:52 file.txt


  • - ← tipo de archivo
  • rwx ← permisos de propietario
  • r-- ← permisos de grupo
  • r-- ← permisos de otros
  • 2 ← numero de hardlinks
  • root ← dueño del archivo
  • root ← grupo del archivo
  • 320 ← tamaño del archivo en bytes
  • dic 2 14:52 ← fecha y hora de la última modificación del archivo
  • file.txt ← nombre del archivo

File System

/bin /boot /dev /etc /home /initrd /lib /lost+found /media /mnt /opt /proc /root /sbin /srv /sys /tmp /usr /var
aplicaciones binarias importantes ficheros de configuración del arranque, núcleos y otros ficheros necesarios para el arranque (boot) del equipo los ficheros de dispositivo ficheros de configuración, scripts de arranque, etc directorios personales (home) para los diferentes usuarios usado cuando se crea un proceso de arranque initrd personalizado librerías del sistema proporciona un sistema de "perdido+encontrado" (lost+found) para los ficheros que existen debajo del directorio raíz (/) particiones montadas (cargadas) automáticamente en el disco duro y medios (media) extraíbles como CDs, cámaras digitales, etc. sistemas de archivos montados manualmente en el disco duro proporciona una ubicación donde instalar aplicaciones opcionales (de terceros) directorio dinámico especial que mantiene información sobre el estado del sistema, incluyendo los procesos actualmente en ejecución directorio personal del usuario root (superusuario); también llamado "barra-root" binarios importantes del sistema puede contener archivos que se sirven a otros sistemas archivos del sistema archivos temporales aplicaciones y archivos a los que puede acceder la mayoría de los usuarios archivos variables como archivos de registros y bases de datos
